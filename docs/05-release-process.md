# Release Process

## Flow

1. Feature branch / task change
2. Local/dev checks
3. Staging deployment
4. Luke/Tony review
5. Production deployment
6. Changelog notes

## Rules

- No production release without approval.
- Every release should have a clear summary of what changed.
- Local/dev checks should pass before staging.
- Staging should be reviewed before production deployment.
- Changelog notes should capture meaningful user-facing and operational changes.
